# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Static Astro site (SSG, no UI framework) for practicing Colombia's B1 driver's license theory exam. All content and UI text is in Spanish; source files consistently omit accent marks in strings and comments — follow that convention.

## Commands

Package manager is **pnpm** (pinned via `packageManager`; Node 22, see `.nvmrc`).

```bash
pnpm run dev          # dev server at http://localhost:4321
pnpm run build        # production build to /dist (fully static HTML/CSS/JS)
pnpm run preview      # serve the production build
pnpm run check        # astro check — type-checks .astro and .ts
pnpm run lint         # ESLint (flat config + eslint-plugin-astro)
pnpm run format       # Prettier --write   (format:check for CI)
pnpm run test         # Vitest — quiz-core logic + Zod data validation
pnpm run validate     # check + lint + format:check + test (full local gate)
```

CI (`.github/workflows/ci.yml`) runs check + lint + format:check + test + build on every PR and push to `main`. Husky hooks enforce lint-staged (pre-commit) and Conventional Commits (commit-msg). See `CONTRIBUTING.md` for the Git Flow branch/commit conventions and `docs/ARCHITECTURE.md` for the full architecture.

TypeScript is strict (`astro/tsconfigs/strict` plus `noUncheckedIndexedAccess` and `verbatimModuleSyntax`).

## Architecture

Data flows one way: typed data files → prerendered page → JSON embedded in HTML → client island.

1. **`src/data/`** — single source of truth. `types.ts` defines `Question`, `ExamMode`, `QuizConfig`. `questions.ts` holds the full question bank (~96 questions). `modes.ts` defines the 7 exam modes (4 CALE groups, mecanico, casos, simulacro) and which question groups each includes.
2. **`src/pages/examen/[modo].astro`** — one dynamic route prerenders a page per mode via `getStaticPaths`. It serializes a `QuizConfig` into `data-config` on `#quiz-root` and the question payload into an inline `<script type="application/json">`.
3. **`src/scripts/`** — the client-side quiz island, split for testability:
   - **`quiz-core.ts`** — pure, DOM-free logic (Fisher-Yates `shuffle`, `prepareQuestion`, `sampleQuestions`, `scoreAnswers`, `formatTime`, `isPassing`). All randomness takes an injectable `rng` so it is deterministically unit-tested (`quiz-core.test.ts`).
   - **`quiz.ts`** — the DOM engine. Parses the embedded JSON, imperatively renders the DOM (wall-clock timer, focus management, persistent `aria-live` region), and handles practice mode (immediate feedback) vs. simulacro mode (timer, no feedback until the end, 80% pass threshold). State lives in memory only — no localStorage. It delegates logic to `quiz-core.ts` and imports `Question`/`QuizConfig` types from `src/data/types.ts`; the payload built in `[modo].astro` must keep matching `Question`'s fields. `initQuiz` returns a teardown (AbortController) so View Transitions don't stack listeners.

**Scoping gotcha:** the quiz's CSS lives in the `<style is:global>` block of `[modo].astro` (classes like `.quiz__*`, `.opt`, `.result__*` are generated by the island, not by Astro markup). It MUST stay `is:global` — Astro-scoped styles never match runtime-created DOM, which renders the whole quiz unstyled. Design tokens (colors, fonts, `--accent` per mode) are CSS variables in `src/styles/global.css`; modes pick an accent via `data-accent`. Color convention: `--signal-amber` is decorative-only (stripes, borders, backgrounds); use `--signal-amber-dark` for amber text and focus outlines (WCAG AA contrast).

## Adding or editing questions

Edit only `src/data/questions.ts`. ID conventions: `g1-NN`…`g4-NN` for groups I–IV, `mec-NN`, `caso-NN`. `answer` is the 0-based index into `options` in their stored (natural) order — the island shuffles options at runtime, so never hand-shuffle them in the data. Counts per mode and the simulacro pool recalculate automatically.

## Content constraints

Questions and `explanation` fields cite Colombian traffic law (Ley 769/2002 and reforms, Ley 2251/2022, Resolucion 20253040037125 de 2025 on CALE). When editing or adding questions, keep explanations grounded in the cited normativa.
